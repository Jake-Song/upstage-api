# 평가 기준

이 문서는 저장소의 세 가지 핵심 워크플로인 채팅 추론, 문서 디지털화,
정보 추출의 평가 기준을 정의한다. 개별 예시만 평가하지 말고, 실제 입력을
대표하며 사람이 검증한 테스트 세트를 기준으로 각 워크플로를 평가한다.

## 공통 평가 원칙

- 테스트 세트는 개발에 사용한 프롬프트 및 예시와 분리한다.
- 일반 입력, 고난도 입력, 비정상 입력을 포함한다. 문서 작업의 경우 문서
  종류, 레이아웃, 언어, 스캔 품질도 다양하게 구성한다.
- 모든 테스트 사례에 대해 사람이 검증한 정답, 전사본 또는 추출 결과를
  준비한다.
- 모델이나 버전을 비교할 때는 동일한 입력과 설정을 사용한다.
- 평균뿐 아니라 실패 건수와 입력 유형별 결과도 함께 보고한다.
- 평균 점수가 기준을 통과하더라도 치명적 오류가 있으면 출시를 차단한다.

사람의 판단이 필요한 항목은 다음의 0-4점 척도를 사용한다.

| 점수 | 의미 |
| --- | --- |
| 4 | 완전히 정확하며 유의미한 문제가 없음 |
| 3 | 전반적으로 정확하며 결과에 영향을 주지 않는 사소한 문제만 있음 |
| 2 | 부분적으로 정확하며 눈에 띄는 누락 또는 오류가 있음 |
| 1 | 대부분 부정확하며 활용할 수 있는 내용이 제한적임 |
| 0 | 부정확하거나 근거가 없고, 사용할 수 없거나 결과가 누락됨 |

가중치를 적용하기 전에 `점수 / 4 * 100`으로 백분율 점수로 변환한다.

## 채팅 추론

최종 `answer`와 반환된 `reasoning`을 모두 평가한다. 추론 과정이 정답의
표현이나 풀이 순서와 같을 필요는 없지만, 타당하고 관련성 있으며 내부적으로
일관된 추론을 통해 답에 도달해야 한다.

| 평가 항목 | 가중치 | 확인 사항 |
| --- | ---: | --- |
| 답변 정확성 | 50% | 최종 결론, 계산 또는 권고가 검증된 정답과 일치하는가 |
| 추론 타당성 | 25% | 핵심 단계가 논리적으로 타당하고, 필요한 가정을 명시하며, 단계 사이에 모순이 없는가 |
| 지시 이행 | 15% | 요청된 제약을 모두 따르고 프롬프트의 모든 질문에 답했는가 |
| 명확성과 관련성 | 10% | 이해하기 쉽고 충분히 설명되며, 불필요하거나 관련 없는 내용이 없는가 |

정답이 명확한 문제는 가중 점수와 함께 정확 일치 정확도도 기록한다. 개방형
문제는 두 명 이상의 평가자가 검토하고, 평가 점수 차이가 1점을 초과하면
합의를 통해 조정한다.

치명적 오류에는 확신에 찬 오답, 조작된 근거, 안전하지 않은 조언, 핵심
사실 또는 논리 오류에 의존한 추론이 포함된다.

권장 초기 출시 기준:

- 가중 점수 85/100 이상.
- 정답이 명확한 문제의 최종 답변 정확도 95% 이상.
- 해결되지 않은 치명적 오류가 없어야 함.

## 문서 디지털화

생성된 Markdown을 원본 문서 및 검증된 기준 전사본과 비교한다. 해당 문서에
적용되는 항목만 평가한다. 표나 특정 구조 요소가 없는 문서는 해당 항목을
제외하고 나머지 가중치의 합이 100%가 되도록 정규화한다.

| 평가 항목 | 가중치 | 확인 사항 |
| --- | ---: | --- |
| 텍스트 충실도 | 40% | 문자, 단어, 문장 부호, 기호, 숫자가 누락이나 중복 없이 정확하게 전사되었는가 |
| 읽기 순서와 레이아웃 | 20% | 문단, 단, 머리글, 바닥글, 페이지 전환이 의도한 읽기 순서로 배치되었는가 |
| 표 충실도 | 20% | 행, 열, 병합 셀, 헤더, 셀 값이 원본 표의 의미를 보존하는가 |
| 구조 보존 | 10% | 제목, 목록, 캡션, 수식, 링크 등 의미 있는 요소가 적절히 표현되었는가 |
| Markdown 품질 | 10% | 깨진 문법이나 불필요한 부산물 없이 유효하고 읽기 쉬운 Markdown인가 |

모든 문서에 대해 문자 오류율(CER)을 보고한다. 단어 경계가 의미 있는 경우
단어 오류율(WER)을 보고하고, 표가 있는 문서는 표 셀 단위 정밀도, 재현율,
F1을 보고한다. 자동 지표 계산 전 줄바꿈 형식처럼 의미에 영향을 주지 않는
차이는 정규화할 수 있다. 단, 의미 있는 문장 부호, 부호, 소수점 구분자,
식별자는 정규화하지 않는다.

치명적 오류에는 페이지나 주요 구역의 누락, 중요한 숫자 또는 부호의 변경,
의미가 바뀌는 읽기 순서 오류, 사용할 수 없는 Markdown 출력이 포함된다.

권장 초기 출시 기준:

- 가중 점수 90/100 이상.
- 문자 오류율이 디지털 원본 문서는 1% 이하, 스캔 문서는 5% 이하.
- 표가 있는 경우 표 셀 F1 90% 이상.
- 페이지 또는 주요 구역 누락을 포함한 해결되지 않은 치명적 오류가 없어야 함.

## 정보 추출

요청에 사용한 정확한 JSON Schema와 사람이 검증한 기준 JSON 객체를 사용해
평가한다. 개별 필드를 채점하기 전에 출력이 스키마를 준수하는지 검증한다.

| 평가 항목 | 가중치 | 확인 사항 |
| --- | ---: | --- |
| 필드 정확성 | 50% | 허용된 정규화 후 추출 값이 문서 및 기준 값과 일치하는가 |
| 완전성 | 20% | 문서에서 확인할 수 있는 모든 필드, 특히 필수 필드가 반환되었는가 |
| 근거성 | 15% | 값이 원문에 근거하며, 문서에 없는 정보를 생성하지 않았는가 |
| 스키마 및 타입 준수 | 15% | 유효한 JSON이며 이름, 타입, 배열, 필수 필드, enum, 형식이 스키마를 따르는가 |

식별자, 이름, 날짜, 금액 등 형식이 중요한 필드는 정확 일치를 사용한다.
값의 의미를 보존하는 경우에만 문서화된 규칙에 따라 공백, 대소문자, Unicode,
날짜 또는 숫자 형식을 정규화할 수 있다. 배열은 요소 단위로 비교하고 중첩
객체는 재귀적으로 필드 단위 비교를 수행한다.

필수 필드 정확도, 전체 필드 정밀도, 재현율, micro F1, 스키마 유효 응답률,
근거 없는 값의 비율을 보고한다. 잘못된 값은 거짓 양성 1건과 거짓 음성
1건으로 모두 계산한다. 누락, 추가 또는 잘못된 타입의 값을 평가에서 임의로
제외하지 않는다.

치명적 오류에는 유효하지 않은 JSON, 스키마를 위반한 출력, 필수 필드 누락,
중요한 값의 생성, 금액·날짜·식별자·당사자의 중대한 오류가 포함된다.

권장 초기 출시 기준:

- 유효한 JSON 및 스키마 준수율 100%.
- 필수 필드 정확도 95% 이상.
- 전체 필드 micro F1 90% 이상.
- 근거 없는 값의 비율 1% 이하.
- 해결되지 않은 치명적 오류가 없어야 함.

## 평가 보고서

각 평가 보고서에는 다음 내용을 포함한다.

1. 모델, API, 프롬프트, 스키마, 코드 버전.
2. 테스트 세트 크기와 작업 및 난이도별 분포.
3. 각 워크플로의 종합 점수와 자동 평가 지표.
4. 입력 유형, 언어, 레이아웃, 스캔 품질별 세부 결과.
5. 치명적 오류와 대표 오류 사례.
6. 이전 승인 기준선과의 비교.
7. 최종 통과 또는 실패 결정과 그 이유.

위 수치는 권장 초기 기준이다. 안정적인 첫 기준선이 만들어진 후에는 제품의
위험 수준에 맞게 조정할 수 있다. 시간에 따른 결과 비교가 가능하도록 모든
기준 변경 내용을 문서화한다.
